# Автотесты для интернет-магазина Wildberries

## Краткое описание

Проект представляет собой набор автоматизированных UI-тестов для интернет-магазина Wildberries (https://www.wildberries.ru/). Реализован полный end-to-end сценарий: от открытия главной страницы до оформления заказа с проверкой цен и применением фильтров.

Проект выполнен на языке Python с использованием паттерна Page Object Model (POM) для обеспечения удобства поддержки и масштабирования тестов.

---

## Функциональность (Что проверяется)

Тест `test_change_headphones` проверяет сквозной сценарий покупки наушников:

| Шаг | Действие | Проверка |
|-----|----------|----------|
| 1 | Открытие главной страницы Wildberries | URL соответствует ожидаемому |
| 2 | Переход в каталог: Электроника → Гарнитуры и наушники | Открыта страница с наушниками |
| 3 | Применение фильтров (цена 2500-50000 ₽, производитель SONY, оригинальный товар, черный цвет) | Фильтры применены |
| 4 | Проверка, что товары производителя SONY присутствуют на странице | Производитель встречается >10 раз |
| 5 | Выбор конкретной модели (SONY WH-1000XM6) | Открыта страница выбранной модели |
| 6 | Нажатие кнопки «Купить» и переход в корзину | Цена в корзине совпадает с ценой в карточке товара |
| 7 | Оформление заказа | Заказ успешно оформлен |

**Дополнительно:** На ключевых шагах выполняются скриншоты, сохраняемые в папку `screens/`.

Особенности реализации
1. Использование undetected-chromedriver
Драйвер undetected-chromedriver позволяет обходить защиту Wildberries от автоматизированных действий, что делает тесты более стабильными.

2. Обработка ошибок
В методе click_menu_button() реализована логика перезагрузки страницы при возникновении ошибки клика.

3. Проверка работы фильтра
В sorted_page.py реализована нестандартная проверка: подсчет количества товаров нужного производителя (>10), что подтверждает корректную работу фильтра.

4. Скриншоты
Скриншоты создаются на следующих этапах:

После применения всех фильтров

После оформления заказа

5. Явные ожидания
Во всех Page Object классах используется WebDriverWait с таймаутами от 10 до 65 секунд для стабильной работы с динамическими элементами.

Возможные проблемы и пути решения
Проблема					Решение
Не открывается страница Wildberries		Проверьте интернет-соединение. Антидетект-защита может блокировать запросы — попробуйте обновить undetected-chromedriver
Товар SONY WH-1000XM6 отсутствует в продаже	Замените модель в тесте на актуальную (файл test_change_product.py, строка "WH-1000XM6")
Не сохраняются скриншоты			Проверьте права на запись в папку screens/ или измените путь в base_class.py
Долгое выполнение теста (таймауты 65 сек)	Таймауты можно уменьшить в файлах Page Object классов (параметр WebDriverWait(driver, X))
